# Blender Scripts — Ellensburg Golf Club OPCD Build

These scripts are part of the OPCD (Open Platform Course Designer) pipeline for building Ellensburg Golf Club as a GS Pro course.

**Pipeline position:** LIDAR data → QGIS processing → heightmap PNG → **Blender (these scripts)** → Unity → GreenKeeper → publish

## Prerequisites

- **Blender 3.6 LTS** or **Blender 4.x** (tested with 3.6 and 4.1)
- A processed heightmap PNG from the LIDAR/QGIS pipeline (16-bit grayscale)
- The metadata JSON from the LIDAR pipeline (generated alongside the heightmap)
- OPCD addon installed in Blender (for final export to Unity/GS Pro)

## Files

| File | Purpose |
|------|---------|
| `import_dem.py` | Imports the heightmap PNG as a terrain mesh with correct real-world scale |
| `materials.py` | Creates PBR material templates for all golf course surface types |
| `vegetation.json` | Estimated tree/vegetation positions (134 trees) for placement |

## Workflow

### Step 1: Import Terrain (`import_dem.py`)

This script creates the base terrain mesh from a heightmap PNG.

1. Open Blender (start a new file or use a clean scene)
2. Switch to the **Scripting** workspace (top tab bar)
3. Click **Open** in the text editor and navigate to `import_dem.py`
4. Click **Run Script** (play button, or Alt+P)
5. A file dialog opens — select your heightmap PNG
6. The script automatically looks for a metadata JSON in the same directory

**What it creates:**
- A subdivided plane mesh ("Terrain") scaled to real-world dimensions
- Heightmap applied as displacement (elevation data drives vertex Z positions)
- An elevation-colored preview material (green gradient)
- A 100m reference grid for scale verification
- Red scale marker cubes at known positions
- An overhead orthographic camera
- A sun light at late-morning angle

**Metadata JSON:** The script looks for `<heightmap_name>_metadata.json` alongside the PNG. This file is generated by the LIDAR processing pipeline and contains `real_width_m`, `real_height_m`, `elevation_min`, `elevation_max`, and `pixel_size`. If not found, it uses defaults for Ellensburg GC (~800x600m, 460-470m elevation).

**Command-line usage:**
```bash
blender --python import_dem.py -- /path/to/heightmap.png
```

**Configuration options** (edit at top of script):
- `SUBDIVISION_OVERRIDE`: Force a specific subdivision level (default: auto-calculated from image size)
- `VERTICAL_EXAGGERATION`: Set > 1.0 to make subtle elevation changes visible during modeling. Reset to 1.0 before export.
- `SCALE_FACTOR`: Adjusts Blender units per meter (default: 1.0, meaning 1 BU = 1 meter)

### Step 2: Apply Materials (`materials.py`)

This script creates PBR-ready materials for all golf course surface types.

1. With the terrain imported, open `materials.py` in the Scripting workspace
2. Run the script — all materials are created in Blender's material browser
3. Select objects and apply materials using any of these methods:
   - **Menu:** Object > Apply Golf Material (opens a dropdown to pick the material)
   - **Script console:** `apply_material_to_selected('Fairway')`
   - **By feature type:** `apply_material_by_feature_type('bunker')` (handles name variations)

**Materials created:**

| Category | Materials |
|----------|-----------|
| Playing surfaces | Fairway, Green, Rough, HeavyRough, TeeBox, FringeCollar |
| Hazards | Bunker, Water |
| Hardscape | CartPath, DirtPath |
| Vegetation | TreeTrunk_Ponderosa, TreeTrunk_Cottonwood, Foliage_Pine, Foliage_Cottonwood, Foliage_Willow |
| Structures | Clubhouse_Walls, Clubhouse_Roof, Concrete |

All materials use the **Principled BSDF** shader with:
- Correct roughness values (smooth greens vs. rough sand)
- Procedural noise for surface texture variation
- Subsurface scattering for grass translucency
- Bump mapping for surface detail
- Colors tuned for semi-arid Central WA environment

Materials are compatible with Unity export and GS Pro rendering.

### Step 3: Place Vegetation (`vegetation.json`)

The vegetation file contains estimated positions for 134 trees across the course. Use this as a starting point — refine positions after on-course GPS data collection.

**Species breakdown:**
- **72 Ponderosa pines** — fairway edges (especially holes 4, 5, 7, 8) and perimeter
- **19 Black cottonwoods** — Yakima River corridor (south/east boundary)
- **21 Pacific willows** — river bank understory
- **22 Ornamental deciduous** — clubhouse area, near tees and greens

**To use in Blender:**
1. Import tree models (from OPCD asset library or SpeedTree exports)
2. Use a script to read `vegetation.json` and place instances at each position
3. Convert lat/lon to local coordinates using the same CRS transform as the terrain
4. Apply appropriate materials from `materials.py` (trunk + foliage per species)

**Coordinate conversion note:** The JSON uses WGS84 lat/lon coordinates. Your terrain is in a local projected CRS (likely UTM Zone 10N, EPSG:32610). You need to convert coordinates using the same transform used for the heightmap. The QGIS project file has this transform defined.

## Tips

- **Save frequently.** Blender with high-poly terrain can be memory-intensive.
- **Use the vertical exaggeration** setting in `import_dem.py` during green shaping. The 10m elevation range at Ellensburg GC is subtle at 1:1 scale.
- **Check the reference grid** after import. The red markers should be at 0m, 100m, and the terrain extent. If they look wrong, check your metadata JSON.
- **Material Preview mode** (Z > Material Preview, or the sphere icons in viewport header) shows the materials without full rendering.
- **For OPCD export**, follow the OPCD documentation for required naming conventions and layer organization.

## Troubleshooting

**Terrain is flat / no displacement visible:**
- Check that the heightmap is 16-bit (8-bit may not have enough precision for the ~10m elevation range)
- Try increasing `VERTICAL_EXAGGERATION` to 3.0 or 5.0
- Verify the metadata JSON has correct `elevation_min` and `elevation_max` values
- Try the vertex displacement method: set `use_vertex_displacement=True` in the file dialog options

**Blender is slow / runs out of memory:**
- Reduce subdivision level (set `SUBDIVISION_OVERRIDE = 10` or lower)
- Close other applications to free RAM
- A 4096x4096 heightmap at full subdivision needs ~4GB RAM for the mesh alone

**Materials look wrong in viewport:**
- Switch to Material Preview mode (not Solid mode)
- Check that the material is assigned to the correct mesh
- Some material features (subsurface, transmission) only render fully in Cycles; they approximate in Eevee

**Can't find the materials after running the script:**
- Open the Material Properties panel (sphere icon in Properties sidebar)
- Click the material dropdown — all created materials are listed
- Or use the operator: Object menu > Apply Golf Material
